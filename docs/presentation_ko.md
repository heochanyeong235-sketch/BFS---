# 발표 구성(논문형 발표 슬라이드 아웃라인)

## 1. 제목/문제정의
- “루빅스 큐브 크로스 최단해 탐색기 개발”
- 왜 크로스인가? (초기 단계, 전체 풀이 품질에 영향)

## 2. 목표
- 정확한 큐브 모델 + 무브 엔진
- 스티커 → 큐비 변환
- 크로스 최단해(축소 상태공간) 계산
- GUI로 사용자가 직접 검증

## 3. 시스템 아키텍처
- core / conversion / ai / visualization / utils / tests
- 데이터 흐름: CubeState → CubeAIState → Cross solver → moves

## 4. 핵심 아이디어
- 전체 큐브 대신 “크로스 엣지 4개”만 상태로 본다
- BFS로 모든 축소 상태의 최단거리 테이블을 만든다
- 실제 풀이는 거리 값을 따라가며 해법 복원

## 5. 구현 디테일(간단)
- 무브 엔진: 스티커 배열 회전 + 인접면 스왑
- 변환: 코너/엣지 식별 및 orientation 판별
- 캐시: 테이블 파일 저장/로딩으로 재실행 가속

## 6. 데모
- GUI에서 Random(20) → 자동 크로스 솔루션 표시
- 무브 버튼으로 솔루션을 한 수씩 적용
- 크로스 완료 여부 확인

## 7. 검증
- 단위 테스트 통과
- 여러 면 크로스 중 최단 해법 선택(동률 출력)

## 8. 한계/향후과제
- 전체 큐브 풀이로 확장(F2L/OLL/PLL)
- 더 나은 UX(예: 3D, 애니메이션)는 범위 외

## 9. 결론
- 축소 상태공간 + BFS 테이블로 크로스 최단해를 빠르게 산출
- 시각화 도구로 개발/검증/학습 루프 완성
